# Proximos pasos

La siguiente iteracion recomendada debe tomar el loop integrado actual como nueva base cerrada: shell, match, pausa, recap y resultado final ya quedaron revalidados. Lo siguiente ya no es seguir corrigiendo el slice M7 minimo, sino abrir trabajo nuevo o validacion manual donde todavia falta evidencia humana.

## Prioridad inmediata

1. Definir el primer slice seguro de `Modo Practica`: libre vs drills, solo o local-multiplayer, y que sistemas valida primero.
2. Revalidar manualmente la shell local con foco en legibilidad, navegacion y claridad para varios jugadores en pantalla compartida.
3. Mantener ownership estricto: `Characters` = identidad, `How to Play` = reglas base, `HUD/pausa/resultados` = recordatorios contextuales.
4. Mantener la disciplina de paridad `base/validation` mientras se abra `Modo Practica` o cualquier capa nueva de shell/HUD.
5. Recién despues de cerrar practica inicial, retomar configuraciones de shell mas amplias o polish de resultados.

## Diferidos explicitos de este cierre parcial

- `Modo Practica`
- replay snippets
- post-muerte `FFA`
- extensiones amplias de shell
- expansion futura de roster

## Regla de sesion

- Si la sesion es documental, priorizar ownership de superficies y evitar duplicar copy entre `Characters`, `How to Play`, practica y match.
- Si la sesion es tecnica, no perder disciplina de paridad `base/validation` ni reintroducir metadata de laboratorio en la ruta de jugador.
- Si la sesion es de gameplay, no abrir polish audiovisual ni balance fino antes de cerrar la practica inicial y el siguiente bloque de playtests manuales.
