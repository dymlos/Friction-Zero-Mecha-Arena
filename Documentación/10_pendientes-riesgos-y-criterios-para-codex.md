# 10 — Pendientes, riesgos y criterios para Codex

## Pendientes importantes
Todavía faltan cierres concretos para:
- condición exacta de victoria en cada modo
- set inicial de habilidades por arquetipo o personaje
- regla definitiva de post-muerte para FFA
- traducción de físicas a valores jugables
- prioridad real entre sistemas para prototipar primero

## Riesgos de diseño
1. Exceso de ruido visual.
2. Sistemas demasiado profundos demasiado pronto.
3. Skills que opaquen el núcleo de choques.
4. Controles avanzados que espanten a jugadores nuevos si no existe una experiencia accesible sólida.
5. Modo Team vs Team demasiado dependiente de coordinación perfecta.
6. FFA desbalanceado si algunos roles necesitan demasiada sinergia externa.

## Criterios para cualquier propuesta de Codex
Cualquier propuesta debería respetar estas reglas:
- preservar la fantasía principal: patinar y chocar con precisión
- validar pronto la sensación de movimiento y choque
- evitar sobreingeniería en etapas tempranas
- priorizar legibilidad
- permitir prototipos jugables después de cada fase
- preferir soluciones modulares y fáciles de entender en Godot
- mantener el diseño accesible para una persona con poca experiencia en programación

## Cómo debería ayudar Codex
Codex debería:
- cuestionar decisiones que dañen claridad o identidad
- proponer fases de implementación progresivas
- evitar suponer mecánicas no cerradas como definitivas
- separar lo central del juego de las capas avanzadas
- usar esta documentación como contexto estable antes de tocar código o arquitectura
